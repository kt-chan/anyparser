**Role:** Senior Backend Engineer & AI Infrastructure Expert.
**Task:** Develop a FastAPI service that serves as a high-performance wrapper for the **MinerU Python SDK**.

### 1. The Core Implementation Logic (CRITICAL)

* **Local Execution, Remote Inference:** The FastAPI service runs **locally**. You must use the `mineru` Python SDK installed in the local environment.
* **SDK Mode:** Initialize the MinerU PDF parser using `vlm-http-client` mode.
* **Connection:** The SDK must point to the VLLM Inference Server at `http://172.20.0.10:8000/v1`.
* **Model ID:** Use `MinerU-2.5`.
* **NO Remote CLI:** Do **not** use `paramiko` to run `mineru` commands via SSH. The SDK itself handles the heavy lifting by sending image/tensor data to the remote VLLM.

### 3. Feature Requirements

* **OpenAI-Compatible Ingestion:** Create a POST endpoint (e.g., `/v1/chat/completions` or a custom `/v1/parse`) that accepts binary PDF data.
* **MinerU Processing:** 1. Receive PDF bytes -> Save to local `temp/`.
2. Invoke MinerU SDK locally using the `vlm-http-client` configuration.
3. The SDK will automatically communicate with the remote VLLM at `172.20.0.10`.
* **Output Packaging:** 1. Locate the local output directory generated by the SDK.
2. Compress the entire directory into a `.tar` archive.
3. Return the archive as a `FileResponse`, then trigger a background task to delete the local `temp/` files.

### 4. Constraints & Environment

* **Windows/PowerShell:** Ensure all path handling uses `pathlib` to avoid backslash/forward-slash issues between Windows (Local) and the expected format.
* **No taskkill:** Manage process lifecycles using Python’s `contextlib` or `try/finally` blocks.
* **SSH Usage:** SSH should **only** be used if you need to verify the remote VLLM status or fetch specific remote logs. It is **not** for running the extraction command.
* **Backend Model Info:** {"id":"MinerU-2.5","object":"model","root":"/model/MinerU-2.5"}

* `test_vllm_connectivity`: Verify the local service can reach `http://172.20.0.10:8000/v1/models`.
* `test_pdf_to_markdown`: A full integration test using a sample PDF.
