### Task 2

**Role:** You are Senior AI Engineer specializing in Vision-Language Models (VLM) and Document Processing.

**Task:** Develop a Python service/module to enrich Markdown files generated by MinerU. The service must identify empty image tags, analyze the images using a remote Visual language model (VLM), and update the Markdown with descriptive alt-text and a "Contextual Description" block. You should scan the @app directory and project root directory .env to understand the current pdf parsing implementation.

### 1. The Enrichment Logic

For every image reference found in the Markdown (e.g., `![](images/abc.jpg)`):

1. **Context Extraction:** Extract up to 500 characters of text appearing *before* and *after* the image tag to provide the VLM with context.
2. **VLM Inference:** Send the image (as base64 or file path) and the text context to the OpenAI-compatible VLM endpoint by taking environment variables (VLM_HOST_PATH, VLM_MODEL_NAME, and VLM_API_KEY) in project root directory .env files.
3. **Prompting:** Use the following system prompt for the VLM:
> "You are a document analysis assistant. Analyze the provided image based on the surrounding text context. Return a JSON object with two fields: 'title' (a 10-word technical alt-text) and 'analysis' (a detailed technical description of the image content for a RAG system)."
4. **Transformation:**
* **Alt-Text:** Fill the empty brackets: `![{title}]({path})`.
* **Context Block:** Append a blockquote immediately below the image: `> **Contextual Description:** {analysis}`.



### 2. Technical Requirements

* **Concurrency:** Use `asyncio` and `httpx` (or `openai` python library) to process images in parallel. Use a `Semaphore` (limit=5) to avoid overloading the remote VLLM service.
* **Regex Processing:** Use a robust regex to find and replace tags without corrupting the rest of the Markdown document.
* **Path Handling:** Use `pathlib`. Ensure the service correctly locates images in the `images/` subfolder relative to the `.md` file.
* **Error Handling:** If the VLM fails or the image is missing, keep the original tag and log the error in `/logs/vlm_enrichment_service.log`.

### 3. Implementation Plan

1. **Service:** Create `app/services/vlm_enrichment_service.py`.
2. **Utility:** Add `app/utils/vlm_client.py` to handle communication with the remote api endpoints.
3. **Integration:** Hook this service into the existing FastAPI flow so that enrichment happens after MinerU finishes but *before* the files are tarred.

### 4. Quality Assurance (Tests)

Write the following tests in `/tests/test_enrichment.py`:

* `test_regex_extraction`: Ensure the code correctly identifies `![](images/...)` patterns.
* `test_context_window`: Verify that the 500-character prefix/suffix text is correctly captured.
* `test_vlm_json_parsing`: Verify the logic correctly handles and retries if the VLM returns non-JSON text.
* `test_markdown_integrity`: Ensure the final `.md` file structure is valid and the context block is correctly placed.

### 5. Reference Environment

* **VLM Endpoint:**  scan environment variables (VLM_HOST_PATH, VLM_MODEL_NAME, and VLM_API_KEY) in project root directory .env files.
* **Local Env:** Windows (PowerShell), Python 3.10+.
